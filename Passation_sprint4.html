Sprint 4 : Passation Défense

Rappel des objectifs du sprint

  • Priorité 1 : Faire des recherches sur le SEO des robots
  • Priorité 2 : Ecrire un programme permettant de récupérer les données à analyser dans les requêtes ainsi que dans les mails reçus

Traitement des données

Afin d'avoir des données simplement utilisables et exploitables, nous avons décidé de les répertorier dans un fichier .csv, qui peut également être ouvert avec notre cher Excel, pour une meilleure visibilité.

Nous avons à notre disposition deux fichiers excel :

  • un fichier nommé requetes.csv : qui recense les informations relatives aux requêtes, que l'on extrait de la base de donnée sur phpMyAdmin
  • un fichier nommé mails.csv : qui recense les informations relatives aux mails reçus sur nos adresses obfusquées, que l'on extrait directement du serveur mail en ligne de commande

Ces deux fichiers sont créés et édités à l'aide d'un programme java que nous allons détailler par la suite.

NB : Nous avons choisi de créer 2 fichiers, séparant les informations des requêtes et celles des mails, pour une raison précise. En effet, pour une requête donnée avec un identifiant x, il est possible de recevoir plusieurs mails. C'est pourquoi, en créant un tableau Excel répertoriant à la fois les informations contenues dans les requêtes ainsi que celles contenues dans les mails, nous nous retrouvons face à un problème de taille : pour une ligne donnée (identifié par x), on pourrait avoir une infinité de colonnes si beaucoup de mails ont été envoyés à la suite de la requête qui a récupéré l'adresse mail d'identifiant x. Mieux vaut donc avoir deux fichiers excel puis croiser les informations.

Requetes.csv

Voici la démarche pour récupérer les informations relatives aux requêtes:

  1. Se rendre sur phpMyAdmin, sur la base de données, à cette adresse : https://cagiva.emn.fr/phpmyadmin/
  2. L'identifiant est "miel" et le mot de passe se trouve dans vos mails, pour des raisons de confidentialité
  3. Se rendre sur la base de données GENERATION_MAIL, où sont stockées les requêtes
  4. Cliquer sur "Exporter", et choisir le format XML
  5. Enregistrer le fichier GENERATION_MAIL.xml sur votre ordinateur, puis copier le dans le projet java mailParsing : notre programme va maintenant pouvoir en extraire toutes les informations utiles !

requestParsing

Le package qui va nous interesser maintenant est requestParsing, où vous trouverez 3 classes :

  • xmlParse : son but est de parser le fichier GENERATION_MAIL.xml pour en extraire les requêtes
  • fillCSV : son but est d'exploiter ces requêtes afin de remplir notre fichier requêtes.csv (qui est à l'origine un fichier vide dans le projet ; si ce n'est pas la première fois que vous lancez le programme, pas de problème, le fichier sera vidé à chaque exécution du programme !)
  • main : classe de lancement du programme

Pour plus de renseignement sur les méthodes de ces classes, vous pouvez consulter la javadoc qui a été générée ainsi que le code commenté ;)

Voici un aperçu du fichier .csv, que l'on obtient en lançant le main :

id;CONTEXTDOCUMENTROOT;CONTEXT_PREFIX;DOCUMENTROOT;GATEWAYINTERFACE

"1";"/rrs.fs/www/mir-data/publish";" ";"/rrs.fs/www/mir-data/publish";"cgi/1.1";

"2";"/rrs.fs/www/mir-data/publish";" ";"/rrs.fs/www/mir-data/publish";"cgi/1.1";

La première ligne du document contient chaque caractéristique de la requête, qui nous intéresse, et chaque ligne qui suit contient les valeurs de ces éléments, avec en première position l'ID de la requête.

Mails.csv

Voici la démarche pour récupérer les informations relatives aux mails:

  1. Se connecter au réseau emn21
  2. Sur Cygwin, taper la commande suivante : scp -r admin@miel.emn.fr:../../var/mail ~
  3. Le dossier "mail" contenant tous les mails reçus se trouve maintenant dans votre dossier racine d'installation de Cygwin !
  4. Copier le dans le projet java mailParsing

mailParsing

Le package qui va nous intéresser maintenant est mailParsing, qui contient 2 classes :

  • mailParsing : cette classe contient toutes les méthodes permettant de retourner les informations intéressantes contenues dans le mail, telles que la date d'envoi, le destinataire, l'expéditeur etc...
  • CsvMail : son but est de remplir le fichier mails.csv, qui se trouve dans le projet, avec les informations que la classe mailParsing a récupéré.

Pour plus de renseignement sur les méthodes de ces classes, vous pouvez consulter la javadoc qui a été générée ainsi que le code commenté ;)

Voici un aperçu du fichier .csv, que l'on obtient en lançant le main :

id;MAILDESTINATAIRE;MAILEXPEDITEUR;MAILDATE;MAILIP

"117";"bodin.thomas117";"catalin.rusu@businessesproperties.eu";"Mon Jan 04 16:44:44 CET 2016";"94.176.131.99"

"117";"cave.martin117";"catalin.rusu@businessesproperties.eu";"Mon Jan 04 15:53:43 CET 2016";"94.176.131.97"

On remarque que le fichier peut contenir plusieurs lignes avec le même ID : en effet, pour la requête 117 par exemple, un mail a été envoyé à deux adresses différentes, et il est également possible d'envoyer plusieurs mails à une même adresse !
