V/. Plannification

I/. CONTEXTE

La sécurité est un problème majeur dans toute application web. Pour assurer la pérennité des sites web, il est indispensable de protéger les données présentes sur les pages internet. Il existe différents types d’information, celui qui nous intéresse ici sont les adresses mail. En effet, aujourd’hui il existe de nombreux robots pirates capables de récupérer les adresses mail présentes sur les pages. Une fois récupérées, des publicités intempestives sont envoyées sur celles-ci. C’est ce que nous appelons le spam. Pour prévenir le spam, des méthodes d’obfuscation ont été développées au cours de ces dernières années avec une efficacité et une utilisabilité variables.

II/. OBJECTIFS

Notre but est de protéger les adresses email présentes sur le site de l’école. Pour ce faire, nous allons tout d’abord dresser un état de l’ART afin de répertorier les principales méthodes d’obfuscation existantes. Le problème sera traité sous deux angles différents. Le premier point de vue est celui de la défense. Basé sur l’état de l’ART, nous allons créer un pot de miel ce qui nous permettra de récolter des statistiques sur la qualité des méthodes d’obfuscation utilisées. En parallèle, nous nous glisserons dans la peau d’un attaquant, le but étant de concevoir un robot capable de récolter les adresses mail offusquées et ainsi d’améliorer notre défense.

III/. VOCABULAIRE

SPAM : en général, il s’agit d’envois en grande quantité d’emails à des fins publicitaires

Pot de miel : en informatique, c’est un ordinateur ou un programme délibérement laissé dans un état de vulnérabilité destiné à attirer et à piéger les pirates

Obfuscation d’adresse email : le code source est transformé avant la compilation de façon à le rendre illisible pour l’être humain

IV/. PERIMETRE

A). Etat de l'ART

1. Méthodes d'obfuscation des adresses mail

i. Adresse en texte plein sans lien

Une première méthode d’obfuscation des adresses mail dans les pages web est simple, mais efficace : afficher sur la page l’adresse mail en texte plein, mais sans lien. De cette façon, les robots moissonneurs ne peuvent pas tomber sur une balise “mailto”, cliquer automatiquement sur le lien et récupérer l’adresse.

  • Cette méthode respecte donc le critère d’efficacité, même si elle reste marginale puisque certains robots sont aussi capables d’analyser le texte de la page et repérer les adresses mails grâce à des caractères bien particuliers comme les @.

  • En revanche, cette solution souffre d’une faible utilisabilité du point de vue de l’utilisateur puisque celui-ci doit copier-coller l’adresse, puis ouvrir manuellement sa boite mail pour écrire au destinataire.

ii. Encodage des adresses par des chaînes de caractères

Il existe également une méthode plus complexe pour contrer les robots moissonneurs : l’encodage. Elle consiste à remplacer certains caractères, aussi bien spécifiques tels que @ mais aussi les lettres ou chiffres communs, par d’autres chaînes de caractères, directement dans la balise “mailto” de la page html.

  • L’utilisabilité reste optimale dans ce cas là, puisque les navigateurs décodent les chaînes de caractères et affichent en clair la bonne adresse mail.

  • Néanmoins, l’efficacité est limitée puisque les robots peuvent reconnaître les tables de cryptage et dès lors décoder eux aussi les adresses obfusquées : entre 60% et 70% de spam sont évités.

iii. Obfuscation via html et css

Une autre méthode d’obfuscation peut être, au lieu de remplacer des caractères dans le document HTML, de séparer l’adresse mail en plusieurs parties et d’y insérer des commentaires en HTML (par exemple : dupond< !-- Commentaire1--> @< !—Commentaire2-->wanadoo< !—Commentaire3-->.fr). Ainsi le robot copie toute la ligne, supposant que le tout est une adresse mail, et essaie de spammer l’adresse mail qui est erronée.

65% des robots n’arrivent pas à contourner cette méthode.

Une méthode similaire est de coder un type en CSS qui permette de cacher du code. Ainsi une partie erronée est insérée dans l’adresse mail en HTML mais elle est typée comme caché ce qui permet à l’utilisateur de la voir correctement tandis que le robot copie l’adresse mail erronée.

Cette méthode permet d’éviter 90% des spams.

Une autre méthode utilisant du code CSS est d’utiliser un type qui permet à l’utilisateur de voir l’adresse mail correctement tandis qu’elle est écrite en miroir dans le code. Cette méthode empêche 100% des robots de récupérer l’adresse mail.

  • Ces trois méthodes ne sont pas faciles d’utilisation puisque ce n’est pas un lien, l’utilisateur doit tout de même copier et coller l’adresse dans sa boite mail.

iv. L’adresse électronique en tant qu’image

Cette méthode consiste en la transformation de l’adresse en une image. L’adresse est déformée et distordue pour ne pas être lisible facilement par un robot. L'humain peut la lire et l'écrire dans son client mail.

  • La méthode est très efficace selon des études menées en 2008, 2010 et 2012. Les robots ne parviennent pas à lire et à récolter l’adresse.

  • L’utilisabilité est en revanche faible puisqu’il n’y a pas de clic/copier-coller disponible, il faut copier l’adresse à la main.

v. Génération de l’adresse par Javascript

Javascript permet de d’obtenir des méthodes d’obfuscation très efficaces qui permettent de conserver un confort maximal pour l’utilisateur. Il existe 2 méthodes utilisant JavaScript visant à obfusquer les adresses mail aux robots, la concaténation et le cryptage.

Concaténation :

Pour cette méthode, l’adresse mail est générée dans le code javascript par concaténation de différentes chaînes de caractères, le tout recomposant l’adresse.

Pour l’instant, très peu de robots sont capables de lire dans le code l’adresse à recomposer (~15%) , cependant de plus en plus y parviennent.

Cryptage :

Le cryptage est la méthode la plus sûre pour conserver un utilisabilité maximale de l’adresse électronique. L’adresse mail est entrée de façon cryptée dans le code javascript, la chaîne de caractères contenant l’adresse est ensuite décryptée pour être injectée dans la page.

A ce jour, aucun robot ne réussit à décrypter les chiffres utilisés pour encoder l’adresse obfusquée.

2. Outils d'interprétation des pages web

Nous avons réalisé une comparaison qualitative des deux outils principaux que sont PhantomJS et Selenium.

i. PhantomJS
Avantages
  • Requiert moins de ressources
  • Ne requiert aucun service extérieur
  • Indépendance : possibilité de lancer le robot via un script en ligne de commande
  • Compile des cas de tests multiples plus vite
  • Convient mieux à des tests simples d'applications web
Inconvénients
  • Ne permet pas d'effectuer des tests sur d'autre navigateurs web
  • Ne convient pas aux tests approfondis d'applications web
  • Manque d'écosystème

ii. Selenium
Avantages
  • Utilise un navigateur, il est donc plus facile de sembler humain
  • Peut être implémenté par plusieurs langages de programmation
  • Fonctionne avec plusieurs navigateurs et plusieurs systèmes d'exploitation
  • Selenium permet un contrôle automatisé de navigateurs réels sur des systèmes réels, assurant ainsi des fonctionnements proches de la réalité et plus visibles
  • Selenium permet de contrôler PhantomJS couvrant de la sorte plus de possibilités
Inconvénients
  • Avec Selenium on ne peut pas lancer plus de quelques navigateurs sur la même machine

B). Défense

Un pot de miel est un espace (serveur, page web, etc.) présentant une vulnérabilité évidente visant à attirer les hackeurs opportunistes et les robots. Il peut avoir deux objectifs différents.

S’il est construit dans un but de recherche, il aura un rôle d’observateur collectant toutes les données possibles sur chacun de ses visiteurs. Ces données serviront ensuite à établir des statistiques.

S’il est construit dans un but de sûreté informatique, le pot de miel servira à attirer les menaces qui attaqueront cette cible vulnérable en premier. Les menaces seront alors identifiées, ainsi que leurs méthodes d’attaque, ce qui permet ensuite de les contrer plus rapidement.

Dans le cas présent, le pot de miel sera une page web attirant des robots moissonneurs qui, par leur présence sur cette page, trahiront leur nature malicieuse, la page n’étant pas conçue pour être accessible au public.

Notre pot de miel sera construit dans un but de recherche.

C). Attaque

Le robot moissonneur, Skynet, que nous allons créer devra répondre à certaines caractéristiques.

Il devra être capable de parcourir notre pot de miel en repérant correctement les adresses mail, c’est-à-dire en contournant les méthodes d’obfuscation. Pour cela, Skynet devra dans un premier temps identifier le fichier robot.txt. Dans ce fichier, si Skynet est cité en tant que “User-Agent” alors il devra sélectionner les pages qui lui sont autorisées ou non (“Allow” ou “Disallow”). Cependant notre robot étant un robot spammer, il tentera d’aller dans toutes les pages même celles qui ne lui sont pas autorisées. Puisqu’en effet c’est dans les pages qui ne sont pas autorisées aux robots, qu’il y aura certainement les adresses mail.

D). Idées

1. Mot de passe à rentrer par l’utilisateur

Cette forme de captcha apparaît lorsque l’internaute clique sur un lien de type “Contactez-moi”, censé ouvrir sa boite mail. Dans ce cas, un captcha avec lettre et/ou chiffre s’affiche et l’utilisateur devra rentrer le code du captcha qu’il voit : c’est alors que l’adresse mail apparaît en hyperlien pour être plus utilisable

2. Réponse à une question

Cette méthode est simple : l’internaute se voit poser une question à laquelle il doit répondre. Par exemple, la réponse demandée peut-être de cocher les images correspondant à un objet.

3. Le code Vigénère

Il s’agit d’une extension du code César utilisé pour l’obfuscation par Javascript. Cela consiste à appliquer le principe du code César (le décalage du caractère dans l’alphabet) en le généralisant à une clé composée de plusieurs caractères utilisée de façon cyclique si elle est plus courte que le message. Chaque caractère de la clé donne le décalage à effectuer pour chiffrer le message (‘a’ représente un décalage de 0, ‘b’ un décalage de 1, etc.). Par exemple, imaginons le message à coder “message code” avec la clé “cle”. m + c = o ; e + l = p ; s + e = w ; s + c = u … Le résultat sera pwulkg nsfp
